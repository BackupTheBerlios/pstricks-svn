\section{Définir la direction où se trouve l'observateur}
C'est le rôle de la commande \texttt{\textbackslash psset\{wiepoint=v$\tt_ x$ v$\tt_
v$ v$\tt_ z$\}}. ({\tt  v$\tt_ x$ v$\tt_ v$ v$\tt_ z$}) sont  les coordonnées
d'un vecteur pointant à partir de l'origine {\sf O} vers l'\oe il de
l'observateur. En réalité il définit la direction de la projection sur le plan
horizontal. La norme du vecteur n'a aucune importance, il est inutile de
prendre un vecteur unitaire. Choisissez simplement les coordonnées en fonction
du point de vue que vous voulez avoir par rapport à l'objet.\\ Dans le dessin
ci-dessous le vecteur $\vec{v}$ a pour coordonnées ({\tt 1 0.5 1.5}). Ce schéma
n'est pas représenté avec ce point de vue, car de ce vecteur nous ne verrions
alors qu'un point : le bout de la flèche ! \\ {\textbf{\textit{ Remarque
:}}{\it  Il ne faut pas choisir l'une des coordonnées égale à 0, cela provoque
une erreur lors de l'affichage de l'image, mais on peut prendre une valeur très
petite : 0.001 par exemple. Il peut y avoir des erreurs provoquées par une
division par 0 dans d'autres rares occasions, il suffit alors de modifier très
légèrement la valeur fautive. }

\section{Choisir le plan sur lequel on veut travailler}
Cette extension permet de tout représenter dans l'espace en 3d, à condition que
ce que l'on souhaite visualiser appartienne à un plan. Si ce n'est pas le cas :
cylindre, sphère etc. il faut découper la surface à représenter en pavés plans
juxtaposés. En fonction de leur nombre, le rendu de la surface sera plus ou
moins réaliste. \begin{description}
\item[\strut{\red Définir le plan} :]  nous disposons d'une commande ({\it qui
est double en réalité}) très efficace : {\tt \textbackslash
ThreeDput[normal=n$\tt_x$ n$\tt_y$ n$\tt_z$]($\Omega\tt_x$ $\Omega\tt_y$
$\Omega\tt_z$)} qui permet à la fois :
\begin{itemize}
  \item de définir mathématiquement le plan par un vecteur normal à ce plan et
  par  un point $\Omega$ appartenant à ce plan.
  \item Ce point $\Omega$ est considéré comme l'origine du repère du système d'axes
   ($\Omega \,X\, Y$), sur lequel tous les repérages des points de ce plan se feront.
   \item Ce repère est automatiquement défini par la commande.
    Il suit les règles d'orientation
   classiques de l'espace que nous allons préciser par la suite.
\end{itemize}

\item[\strut{\red Exemple 1 : représenter les faces visibles d'un cube}]~
\begin{itemize}
  \item face A : {\tt \textbackslash ThreeDput[normal=0 1 0](1,1,0)}
  \item face B : {\tt \textbackslash ThreeDput[normal=1 0 0](1,0,0)}
  \item face de dessus :{\tt \textbackslash ThreeDput[normal=0 0 1](0,0,1)}
\end{itemize}

choisis.\\[1cm]

    \textbf{Quelles sont les règles d'orientation ?}\\
    Par exemple pour la face {\sf B} :\\ $\Omega$ étant le point de cette face
    qui dans le repère de référence O$xyz$ a pour coordonnées (1,0,0).
    $\Omega$\,X$\sf_B$,$\Omega$\,Y$\sf_B$,$\vec{n}\sf_B$ forment un trièdre direct. Plus
pratiquement avec la main droite, on doit avoir (les trois doigts formant un
trièdre) :
\begin{itemize}
  \item pouce = $\Omega$\,X$\sf_B$
  \item index = $\Omega$\,Y$\sf_B$
  \item majeur = $\vec{n}\sf_B$
\end{itemize}
Toutes les faces suivent cette règle. C'est ainsi qu'en respectant cette règle,
je peux écrire sur les faces du cube, de telle sorte qu'en restant à
l'extérieur du cube je peux lire, en gardant les pieds sur le plan horizontal
O$xy$, les mots à l'endroit ({\it les normales sont orientées vers l'extérieur
du cube}).

\begin{itemize}
  \item $0\leq x\leq 1$
  \item $0\leq y\leq 1$
  \item $0\leq z\leq 1$
\end{itemize}

ce plan, c'est le {\it pied} de la perpendiculaire de O à ce plan.\\ %
